# Инструкция по сборке уникальных версий (Unique IPA Build)

Для того чтобы каждая сборка имела уникальный бинарный след (разный вес и идентификатор), необходимо использовать скрипт `build_unique_ipa.py`.

## Что делает скрипт:
1. **Модификация ресурсов**: Добавляет случайное количество байт (padding) в конец файла `GlassAlarm/Resources/crystal.wav`.
2. **Модификация кода**: Обновляет значение `buildUniqueId` в файле `GlassAlarm/App/GlassAlarmApp.swift`.
3. **Сборка**: Автоматически пушит изменения в GitHub и запускает удаленную сборку через GitHub Actions.

## Как запустить сборку:
Запустите следующую команду в терминале:
```powershell
python build_unique_ipa.py
```

## Важные замечания по коду (Swift):
При редактировании файлов `GlassAlarmApp.swift` и `AlarmStore.swift` соблюдайте следующие правила, чтобы не сломать сборку:

1. **Не дублируйте buildUniqueId**: Скрипт сам найдет и заменит строку `private static let buildUniqueId = "..."`. Не добавляйте вторую такую строку вручную.
2. **Использование await в условиях**: В Swift нельзя писать `guard condition || await asyncFunction()`. Вместо этого используйте конструкцию:
   ```swift
   if !condition {
       guard await asyncFunction() else { return }
   }
   ```

## Где найти результат:
После успешного завершения скрипта, готовый `.ipa` файл будет скопирован в папку:
`dist/`
