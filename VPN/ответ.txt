Я разрабатываю iOS VPN-клиент на SwiftUI с использованием Xray Core и Network Extension (NEPacketTunnelProvider).

Сборка выполняется через GitHub Actions на macOS runner с помощью xcodebuild.

Цель: получить IPA для последующей локальной подписи через AltStore/AltServer.

Условия:

* Нет платного Apple Developer аккаунта.
* Нет доступа к Apple Developer Portal.
* Нет Provisioning Profiles для Network Extension.
* Нет возможности открыть Xcode GUI на сервере.
* Сборка должна выполняться полностью через CLI.

Текущая проблема:

xcodebuild завершается с ошибкой Exit Code 65.

Проект содержит:

* Основное приложение (SwiftUI)
* PacketTunnel Extension (NEPacketTunnelProvider)
* Xray Core Framework

В проекте уже добавлены:

* com.apple.developer.networking.networkextension
* App Groups
* необходимые Entitlements

При сборке используются:

CODE_SIGNING_ALLOWED=NO
CODE_SIGNING_REQUIRED=NO
CODE_SIGN_IDENTITY=""
ENTITLEMENTS_ALLOWED=NO

Несмотря на это, Xcode всё равно блокирует сборку таргета Network Extension.

Необходимо определить:

1. Можно ли собрать приложение с PacketTunnel Extension полностью без подписи и без provisioning profile?

2. Какие изменения нужно внести в:

   * project.pbxproj
   * Build Settings
   * Build Phases
   * Entitlements
   * xcodebuild команды

3. Возможно ли временно отключить проверки Network Extension на этапе компиляции, чтобы получить бинарник для последующей подписи через AltStore?

4. Если это невозможно, какая минимальная конфигурация Apple Developer требуется исключительно для сборки PacketTunnel Extension?

5. Какие конкретные настройки используются в open-source VPN клиентах:

   * Streisand
   * Hiddify
   * V2Box
   * FoXray
   * Shadowrocket-подобных проектах

Интересуют конкретные решения:

* команды xcodebuild;
* изменения в project.pbxproj;
* примеры Build Settings;
* обходы проверки Entitlements;
* CI/CD конфигурация GitHub Actions.

Нужен практический способ получить собираемый IPA с Network Extension без ручного вмешательства в Xcode.
